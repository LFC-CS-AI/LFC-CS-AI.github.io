Visitor Analytics

Privacy Notice

Last updated: July 13, 2026

What the public map shows

The homepage shows cumulative visits grouped by country or region. Bright map markers are approximate and never represent a street address or a visitor's exact position. Network routing, VPNs, and proxies can make a country estimate inaccurate.

What is retained

The visitor analytics database stores an ISO two-letter country or region code, a cumulative count, daily aggregate counts, and the time that a country or region was last recorded. It does not store raw IP addresses, names, email addresses, cities, postal codes, latitude or longitude, or street addresses.

These aggregate records have no automatic expiry. They remain available unless the site owner deletes the Cloudflare D1 database or the associated account. A public daily snapshot of the same aggregate totals is also kept in the website repository so the latest retained totals can still be shown during a service outage.

How the country or region is determined

The site sends a request to a Cloudflare Worker. Like other web infrastructure, Cloudflare receives standard network request information and derives an approximate country or region code. The site's Worker writes only that code and aggregate counts to D1; it does not write the incoming IP address to the analytics database. Cloudflare's handling of service data is described in its privacy policy.

How repeat visits are limited

After a visit is recorded, the browser stores only the UTC calendar date in local storage. The page uses that date to avoid adding another visit from the same browser on the same day. Clearing browser storage, using another browser or device, or blocking local storage can change this behavior, so the number is a page-visit measure rather than a guaranteed count of unique people.

Your choice

You can block the analytics request or disable JavaScript. If blocked, the visit is not added; the latest daily aggregate backup may still be displayed on the map.
